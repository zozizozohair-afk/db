# نظام تحديث بيانات العملاء الشامل

## نظرة عامة

تم تطوير نظام شامل لحل مشكلة تحديث بيانات العملاء عبر جميع الجداول المرتبطة في قاعدة البيانات. هذا النظام يضمن تحديث البيانات بشكل متسق ومتزامن عبر جميع الجداول دون أي أخطاء.

## المشكلة التي يحلها النظام

أحياناً تأتي بيانات العميل ناقصة (مثل عدم وجود رقم هوية أو رقم هاتف)، فيتم إدخال أرقام افتراضية. عند تعديل بيانات العميل لاحقاً، كان من الضروري تحديث هذه البيانات في جميع الأماكن المرتبطة:

- جدول العملاء
- جدول العقود
- جدول المعاملات المالية
- جدول الوحدات
- جدول التنازلات
- جدول عقود إعادة البيع

## الملفات المضافة/المعدلة

### 1. `customer_update_service.dart` (جديد)
خدمة شاملة لتحديث بيانات العملاء عبر جميع الجداول المرتبطة.

#### الوظائف الرئيسية:
- `updateCustomerDataEverywhere()`: تحديث بيانات العميل في جميع الجداول
- `checkCustomerDataDistribution()`: فحص توزيع بيانات العميل عبر الجداول
- دوال مساعدة لتحديث كل جدول على حدة

### 2. `العملاء.dart` (معدل)
تم تعديل دالة `saveCustomer()` لاستخدام النظام الجديد عند تعديل بيانات العميل.

#### التحسينات المضافة:
- استخدام `CustomerUpdateService` عند تعديل البيانات
- إضافة زر لعرض تقرير توزيع البيانات
- إضافة دالة `_showCustomerDataDistribution()` لعرض التقرير

### 3. `test_customer_update.dart` (جديد)
صفحة اختبار لتجربة النظام والتأكد من عمله بشكل صحيح.

## كيفية الاستخدام

### 1. تحديث بيانات العميل العادي
```dart
// في صفحة العملاء، عند الضغط على "تعديل"
// سيتم تلقائياً استخدام النظام الجديد إذا تم تغيير:
// - رقم الهوية
// - اسم العميل
```

### 2. عرض تقرير توزيع البيانات
```dart
// في نافذة تعديل العميل، اضغط على أيقونة التحليلات
// لعرض توزيع بيانات العميل عبر الجداول المختلفة
```

### 3. الاستخدام البرمجي
```dart
import 'customer_update_service.dart';

// تحديث بيانات العميل
final results = await CustomerUpdateService.updateCustomerDataEverywhere(
  oldIdentityNumber: 'رقم_الهوية_القديم',
  newCustomerData: {
    'name': 'الاسم_الجديد',
    'identityNumber': 'رقم_الهوية_الجديد',
    'phoneNumber': 'رقم_الهاتف_الجديد',
  },
  context: context,
);

// فحص توزيع البيانات
final distribution = await CustomerUpdateService.checkCustomerDataDistribution(
  'رقم_الهوية'
);
```

## الجداول المدعومة

### 1. جدول العملاء (`customers`)
- `name`: اسم العميل
- `identityNumber`: رقم الهوية
- `phoneNumber`: رقم الهاتف

### 2. جدول العقود (`contracts`)
- `clientName`: اسم العميل
- `identityNumber`: رقم الهوية
- `clientIdentity`: رقم الهوية (حقل بديل)
- `clientData.identityNumber`: رقم الهوية (في البيانات المدمجة)
- `idNumber`: رقم الهوية (حقل بديل)

### 3. جدول المعاملات المالية (`financial_transactions`)
- `customerName`: اسم العميل
- `clientName`: اسم العميل (حقل بديل)
- `idNumber`: رقم الهوية
- `identityNumber`: رقم الهوية (حقل بديل)
- `customerId`: رقم الهوية (حقل بديل)

### 4. جدول الوحدات (`apartments`)
- `customerName`: اسم العميل
- `customerId`: رقم الهوية

### 5. جدول التنازلات (`contract_assignments`)
- `originalOwnerName`: اسم المالك الأصلي
- `originalOwnerID`: رقم هوية المالك الأصلي
- `newOwnerName`: اسم المالك الجديد
- `newOwnerID`: رقم هوية المالك الجديد

### 6. جدول عقود إعادة البيع (`resale_contracts`)
- `clientName`: اسم العميل
- `identityNumber`: رقم الهوية

### 3. الشفافية

- عرض الأخطاء إن وجدت
