# إعداد Firebase Storage لرفع الصكوك

## المشكلة
إذا كانت عملية رفع الصكوك لا تكتمل أو يبقى الزر في حالة التحميل، فالمشكلة غالباً في قواعد Firebase Storage.

## الحل

### 1. إعداد قواعد Firebase Storage

1. اذهب إلى [Firebase Console](https://console.firebase.google.com/)
2. اختر مشروعك `msakin-2b6ae`
3. من القائمة الجانبية، اختر **Storage**
4. اذهب إلى تبويب **Rules**
5. استبدل القواعد الموجودة بالقواعد التالية:

```javascript
rules_version = '2';

service firebase.storage {
  match /b/{bucket}/o {
    // Rules for the 'deeds' folder
    match /deeds/{allPaths=**} {
      // Allow read and write access to authenticated users
      allow read, write: if request.auth != null;
    }
    
    // Default rule for other files
    match /{allPaths=**} {
      // Allow read and write access to authenticated users
      allow read, write: if request.auth != null;
    }
  }
}
```

6. اضغط **Publish** لحفظ القواعد

### 2. التحقق من المصادقة

تأكد من أن المستخدم مسجل دخول قبل محاولة رفع الملفات:

```dart
// في main.dart أو في صفحة تسجيل الدخول
if (FirebaseAuth.instance.currentUser == null) {
  // يجب تسجيل الدخول أولاً
  Navigator.pushReplacementNamed(context, '/login');
}
```

### 3. التحسينات المضافة

#### أ. التحقق من حجم الملف
- الحد الأقصى: 10 ميجابايت
- يتم التحقق قبل بدء الرفع

#### ب. Timeout للعمليات
- رفع الملف: 5 دقائق كحد أقصى
- حفظ البيانات: 30 ثانية كحد أقصى

#### ج. Metadata للملفات
- نوع الملف: `application/pdf`
- معلومات إضافية: رقم الشقة، اسم الملف الأصلي

#### د. مراقبة تقدم الرفع
- عرض نسبة التقدم في Console
- إلغاء العملية عند انتهاء المهلة

### 4. استكشاف الأخطاء

#### أ. خطأ الصلاحيات
```
خطأ في الصلاحيات - تأكد من إعدادات Firebase Storage Rules
```
**الحل:** تطبيق القواعد المذكورة أعلاه

#### ب. خطأ الشبكة
```
خطأ في الشبكة - تأكد من الاتصال بالإنترنت
```
**الحل:** التحقق من الاتصال بالإنترنت

#### ج. انتهاء المهلة
```
انتهت مهلة رفع الملف. يرجى المحاولة مرة أخرى
```
**الحل:** 
- تقليل حجم الملف
- التحقق من سرعة الإنترنت
- المحاولة مرة أخرى

#### د. حجم الملف كبير
```
حجم الملف كبير جداً. يجب أن يكون أقل من 10 ميجابايت
```
**الحل:** ضغط ملف PDF أو تقليل جودته

### 5. اختبار النظام

1. افتح التطبيق في المتصفح
2. اذهب إلى صفحة تحديث الصكوك
3. ابحث عن مشروع
4. اختر ملف PDF (أقل من 10 ميجابايت)
5. اضغط "رفع الصك"
6. راقب Console للرسائل التفصيلية

### 6. ملفات القواعد المحلية

تم إنشاء ملف `storage.rules` في جذر المشروع يحتوي على القواعد المطلوبة. يمكن نشرها باستخدام:

```bash
firebase deploy --only storage
```

### 7. مراقبة الأداء

يمكن مراقبة عمليات الرفع من خلال:
- Firebase Console > Storage > Files
- Firebase Console > Storage > Usage
- Developer Tools > Console (للرسائل التفصيلية)

## ملاحظات مهمة

1. **الأمان:** القواعد الحالية تسمح للمستخدمين المسجلين فقط برفع الملفات
2. **الحجم:** الحد الأقصى 10 ميجابايت لكل ملف
3. **النوع:** ملفات PDF فقط
4. **التسمية:** يتم إضافة timestamp لتجنب تضارب الأسماء
5. **البيانات:** يتم حفظ معلومات الملف في Firestore تلقائياً

## في حالة استمرار المشكلة

1. تحقق من Console للأخطاء التفصيلية
2. تأكد من تسجيل الدخول
3. تحقق من إعدادات Firebase في `firebase_options.dart`
4. راجع قواعد Firestore أيضاً
5. تأكد من تفعيل Firebase Storage في المشروع
